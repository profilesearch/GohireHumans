AI QA report template

A good human QA task should produce evidence, not just an opinion. Use this report format for AI content, RAG answers, support replies, listings, and agent outputs.

1. Summary

  • Pass / needs revision / high risk
  • One-sentence reason
  • Reviewer confidence

2. Issue log

  • Issue type
  • Where it appears
  • Why it matters
  • Suggested correction

3. Evidence

  • Source links
  • Relevant screenshots
  • Claim-level notes
  • Quote or excerpt checked

4. Scores

  • Accuracy: 1–5
  • Completeness: 1–5
  • Tone/brand fit: 1–5
  • Risk: low/medium/high

5. Final recommendation

  • Safe to publish?
  • Needs human owner review?
  • Needs expert/professional review?
  • What to fix first
